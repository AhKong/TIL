## LXC (LinuX Container)

- 단일 리눅스 시스템에 동작하고 있는 프로세스를 격리시켜 각 프로세스마다 독자적인 리눅스 시스템 환경을 구축하는 것
- OS 수준에서의 가상화 라고도함


## 가상화 VS LXC 
- 일반적으로 가상화는 HYPERVISOR라는 논리적 플랫폼을 이용해, 하나의 Host OS( 본래 PC본체에 설치된 OS ) 위에 여러 Guest OS(추가로 구동하고자 목표로하는 OS)를 구동하는 기술
- 리눅스 컨테이너는 `하나의 리눅스 시스템에서 프로세스들을 격리시켜 독자적인 시스템환경을 구축`
- 즉, 리눅스 컨테이너에서 가상화된 프로세스들은 모두 리눅스 시스템을 호환해야 함 -> 이 부분에 대해서는 아직 뭔 말인지 정확하게 납득이 가지 않음 좀 더 조사 후 내용 보충 할 것.

## 리눅스 컨테이너의 장점

- 전체 OS를 설치하는 것이아니기에, 특정 어플리케이션 환경만 따로 간편하게 구축을 할 수 있음
- 구축에 필요한 전체적인 메모리 또한 감소
- 상대적으로 가벼움
- HYPERVISOR를 이용해 가상화를 하지 않기 때문에 오버헤드가 적게 발생함 
- 상태를 가지지 않는다(stateless)
    - 컨테이너가 실행되는 환경은 독립적이라 다른 컨테이너에게 영향을 주지 않음 도커같은 이미지 기반으로 컨테이너를 사용하는 경우특정 실행 환경을 쉽게 재사용이 가능함!
- 프로세스가 동작에 필요한 자원만큼을 사용 -> ?????

>이렇게 때문에 컨테이너를 사용하게 되면 무겁고 느린 가상화를 사용하지 않고도 손쉽고 빠르게 개발 및 배포 가능하다!!


## 컨테이너가 사용하는 기술

- 컨테이너라는 기능이 탄생할 수 있었던 이유는 `리눅스 커널에서 제공하는 기능 덕분`
- 리눅스 컨테이너 공식문서에서는 관련한 기능들
    - `namespace`
    - Apparmor와 SELinux 프로파일
    - Seccomp 정책
    - `Chroots`
    - kernel capability
    - `CGroups`

- Chroots : 프로세스의 루트 디렉토리를 변경하는 것입니다.
 이를 통해 우리는 다른 프로세스가 액세스 할 수있는 디렉토리를 제한할 수 있습니다.
- namespace: 하나의 시스템에서 프로세스를 격리시킬 수 있는 가상화 기술 (각 별개의 독립된 공간을 사용하는 것처럼 격리된 환경을 제공하는 경량 프로세스 가상화 기술)
    - 1) PID namespace 
        - 프로세스에 할당된 고유한 ID를 말하며 이를 통해 프로세스를 격리 시킬 수 있다.
        - namespace가 다른 프로세스는 서로 액세스 할 수 없다.  
    - 2) Network namespace
        - 네트워크 디바이스, IP 주소, port 번호, 라우팅 테이블, 필터링 테이블등의 네트워크 리소스를 namespace마다 격리시켜 독립적으로 가질 수 있다. 
        - 해당 기능을 사용하면 OS상에서 사용중인 PORT가 있더라도 컨테이너 안에서 동이란 port를 사용 가능함
    - 3) UID namespace
        - UID,GID를 namespace별로 독립적으로 가질 수 있도록 한다.
        - namespace 안과 호스트 OS 상에서 서로 다른 UID, GID를 가질 수 있다.
    - 4) Mount namespace
        - 호스트 OS와 namespace가 서로 다른 격리된 파일시스템 트리를 가질 수 있도록 한다.
        - mount : 컴퓨터에 연결된 기기나 기억장치를 OS에 인식시켜 사용가능한 상태로 만드는 것을 뜻함
    - 5) UTS namespace
        - namespace별로 호스트명이나 도메인 명을 독자적으로 가질 수 있다.
    - 6) IPC namespace
        - 프로세스간 통신(IPC) 오브젝트를 namespace별로 독립적으로 가질 수 있다.                       

> __namespace<br>__
VM에서는 각 게스트 머신별 독립적인 공간을 제공하고 서로가 충돌하지 않도록 하는 기능을 갖고 있다. 리눅스에서 동일한 역할을 하는 것이 Namespace이며 커널에 내장되어 있음<br>
Hypervisor는 Hardware Resource를 가상화 하고 위에 올라가는 Guest OS에는 가상화 된 형태의 H/W를 제공하게 되므로 각 Guest OS는 완전히 다른 환경으로 분리된다.
하지만 Namespace의 경우 Hardware Resource 레벨의 가상화가 아니고 동일한 OS와 동일한 Kernel에서 작동하게 된다. 단지 격리된 환경만 제공하는 것
(Namespace는 H/W 자원을 가상화하는 것이 아니라 Linux 내의 자원을 가상화 하는 것)

-  Cgroups(Control groups) : 프로세스의 리소스를 격리하고 사용을 제어하는 기능입니다. Cgroups의 역할은,
    - 자원 제한 : 특정 프로세스의 메모리 사용량을 제한.
    - 우선 순위 : 특정 프로세스에 더 많은 CPU와 디스크 I/O 처리량을 할당.
    - 기록 : 프로세스가 자원을 얼마나 사용하고 있는지 측정.
    - 제어 : 프로세스를 멈추거나, 프로세스의 체크포인트를 설정해주거나 재시작 가능.
